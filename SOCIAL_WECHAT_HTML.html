缠论交易系统 v2.0 —— 从数据采集到回测的完整端到端实战

导读：一个能在 GitHub Actions 上自动跑的 A 股缠论信号平台。本文带你理解设计思路、快速复现运行，并附上演示图与下载指南。

1. 我们的目标（2分钟读懂）

把缠论信号工程化：异步采集（热/全量）、缠论三类买卖点识别（线段/中枢/多周期）、回测与风险管理，以及 GitHub Actions 自动化发布报告。

2. 系统架构一眼看懂

异步采集（aiohttp）→ SQLite (WAL) → 缠论信号引擎 → 回测与风控 → GitHub Actions 自动化。

3. 为什么这么做（可复现的要点）

短句说明：异步提高吞吐；WAL 降低写锁；safe_value 容错字段格式；多周期共振提高置信度；回测结合仓位才可用。

4. 快速上手（直接运行）

本地快速验证（热模式，26 只，约 1 秒）：

python3 run_complete_system.py --mode all --collect-mode hot --no-backtest

完整采集（全量，5000+，约 6–9 分钟）：

python3 run_complete_system.py --mode all --collect-mode all

运行后查看 logs/analysis_report.txt 和 logs/backtest_report.txt

7. FAQ（常见问题）

Q: 采集 0 条怎么办？
A: 用 curl 测试行情源或配置代理，检查 `HTTP_PROXY`/`HTTPS_PROXY`。
Q: 为什么没信号？
A: 缠论对历史深度敏感，建议至少 10–15 条 minute K 线。
Q: GitHub Actions 超时？
A: 可把 `timeout-minutes` 提高到 45 或 60。
